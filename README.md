# Wise: Software Architect Mode

本プロジェクトは、AI開発アシスタント（エージェント）を単なるコーダーではなく、「**シニア・ソフトウェア・アーキテクト**」として動作させるためのカスタムスキル（`wise`）および設定ファイル群です。

システム全体を俯瞰する思考、テスト駆動開発（TDD）、並行処理や状態管理の厳密な設計、そして妥協のない品質管理をAIに強制し、より堅牢で保守性の高いコード生成を実現します。

## 思想・コア原則 (Core Philosophy)

1. **局所的ではなく、システム全体で考える (Think Systemically, Not Locally)**
   - 「どう直すか」ではなく「なぜこのバグが存在するのか」「他のどこに影響するか」を問う。
2. **速度よりも品質を優先する (Quality Over Velocity)**
   - アーキテクトは理解に70%、コーディングに30%の時間を費やす。理解なきコーディングは行わない。
3. **自らのコードの敵となる (Be Your Own Adversary)**
   - 常に「並行実行されたらどうなるか？」「入力がnull/ゼロ/巨大だったらどうなるか？」と自問自答し、エッジケースを検証する。

## リポジトリ構成 (Repository Contents)

本リポジトリには以下のファイルが含まれています。

* **`SKILL.md`**
  * エージェントのコアとなる動作定義書。AIが従うべき8つのフェーズ（理解・設計・TDD・実装・検証・文書化など）と行動規範を定義しています。
* **`PATTERNS.md`**
  * アーキテクトとしてのベストプラクティスを具体化したコード例。TOCTOU（Time-of-Check to Time-of-Use）の防止策や、強力なテストパターン、マジックナンバーの排除などを規定しています。
* **`CHECKLISTS.md`**
  * 開発プロセスの各フェーズでAI（および開発者）が確認すべきクイックリファレンス・チェックリストです。
* **`settings.local.json`**
  * AIエージェントに許可するBashコマンド（`ls`, `cat`, `grep`, `git`, `gh`, `python3`, `which`, `find`）を定義したセキュリティ・実行環境設定ファイルです。
* **`.gitignore`**
  * プロジェクト固有のGit除外設定ファイルです。

## 開発ワークフロー (The 8-Phase Process)

`wise` モードが起動した際、エージェントは以下の厳格な8つのフェーズに従ってタスクを実行します。

1. **Phase 1: Understanding & Planning (理解と計画)**
   - 既存ドキュメントの読み込みと、タスクの複雑度評価、ToDoリストの作成。
2. **Phase 2: Codebase Exploration (コードベースの探索)**
   - 既存パターンの把握と影響範囲の特定（`grep`等での徹底的な裏付け）。
3. **Phase 3: Test-Driven Development (テスト駆動開発)**
   - RED（失敗するテスト）→ GREEN（最小限の実装）→ REFACTOR（リファクタリング）の徹底。
4. **Phase 4: Implementation (実装)**
   - 既存の定数やログ、エラーハンドリングの規約に従ったコーディング。
5. **Phase 5: Test Suite Verification (テストスイートの検証)**
   - 変更範囲に応じたテストの実行とリグレッション（デグレ）の防止。
6. **Phase 6: Documentation & GitHub (ドキュメントとGitHub更新)**
   - ドキュメントの同期とIssueの更新。
7. **Phase 7: Pre-Commit Review (コミット前レビュー)**
   - エッジケース、セキュリティ、アーキテクチャ観点での自己批判的なレビュー。
8. **Phase 8: PR & Review Readiness (PR作成とレビュー準備)**
   - Diffの最終確認と、クリーンなPull Requestの作成。

## 使い方 (Usage)

AIアシスタントに以下の条件を満たす複雑なタスク（3ファイル以上にまたがる変更、新規機能追加、アーキテクチャのリファクタリング、並行処理を伴うバグ修正など）を依頼する際に適用されます。

エージェントは出力の冒頭で `## [WISE MODE]` と宣言し、アーキテクトとしての厳格な基準でタスクの処理を開始します。

※ 50行未満の単一ファイルの修正や軽微な設定変更などの単純なタスクの場合は、簡略化されたフェーズ（1 → 4 → 7）のみが実行されます。
